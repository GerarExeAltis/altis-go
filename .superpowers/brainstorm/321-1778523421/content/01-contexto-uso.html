Onde o jogo Roleta vai acontecer fisicamente?

Essa decisão muda o sentido do QR Code, do "Standby", da autenticação do jogador e da estratégia anti-fraude. Escolha a que melhor descreve o cenário real de uso.

A

Kiosk / Totem em evento ou loja física

Uma tela grande (TV/totem) mostra a roleta em modo Standby. Quando alguém se aproxima, o admin (ou um botão) inicia o ciclo: o totem mostra um QR Code dinâmico → cliente escaneia com o celular → preenche dados no celular → roleta no totem gira → resultado aparece no totem → cliente vai retirar o prêmio com o admin.

Cenário A — Dois dispositivos
TOTEM
Roleta 3D / Standby / QR / Resultado
CELULAR DO CLIENTE
Escaneia QR → preenche dados → confirma
B

Dispositivo pessoal do jogador (sem totem)

Cada jogador acessa a URL direto no próprio celular (provavelmente via QR Code estático impresso em material promocional, banner, embalagem etc.). Cadastra-se, joga a roleta no próprio aparelho, vê o resultado, e depois retira/recebe o prêmio.

Cenário B — Um dispositivo
CELULAR DO JOGADOR
Login → Roleta → Resultado → Comprovante de prêmio
C

Híbrido — totem + celular, mas a roleta gira no celular

Totem só serve como "porta de entrada" mostrando o QR. Cliente escaneia, e todo o fluxo (cadastro + roleta + resultado) acontece no celular dele. O totem só volta a mostrar Standby. Vantagem: roleta 3D otimizada para tela pequena, menos custo de hardware. Desvantagem: experiência menos "espetáculo".
